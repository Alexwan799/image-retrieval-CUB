# Roadmap

> [!note] P1 roadmap
> 
> P1 目标（先定清）
> - 用 ResNet-50 作为 backbone
> - 去掉原分类头，接一个 128-d embedding head
> - 前向输出稳定为 (B, 128)
> - 能进入后续 Triplet Loss 训练
> 
> 第 1 步：先只验证 backbone+head 前向
> - 定义一个 EmbeddingModel（先不管 loss）
> - 用预训练 ResNet-50 权重（自动下载并缓存）
> - 去掉原 fc（保留卷积特征提取部分）
> - 接一个线性层把 2048 -> 128
> - 用随机输入或 dataloader 的一个 batch 测试前向 shape
> 
> 验收标准：
> - 输入 (B, 3, 224, 224)，输出 (B, 128)
> - 不报维度错误
> 
> 第 2 步：决定是否做 L2 normalize（我推荐做）
> - 检索任务常用单位向量 embedding
> - 这样后面 cosine 相似度、triplet 距离更稳定
> 验收标准：
> - 每个 embedding 向量范数接近 1（如果你加了 normalize）
> 
> 第 3 步：参数冻结策略（先简单）
> - 第一版建议：不冻结（全量微调），先跑通闭环
> - 如果显存或收敛不稳，再改成先冻 backbone 前几层
> 验收标准：
> - 反向传播可以正常跑，loss 能下降趋势可见
> 
> 第 4 步：接入 triplet 训练前的必要准备
> - 明确 batch 内标签可用于采样
> - 先实现一个最简 triplet 采样逻辑（别一上来就复杂 mining）
> 验收标准：
> - 能构造有效 triplets（anchor/positive/negative）
> 
> 第 5 步：训练最小闭环
> - 1 个 epoch 小跑（甚至几十个 iter）
> - 打印 loss、保存 checkpoint
> 验收标准：
> - 训练不崩
> - loss 有基本下降
> - checkpoint 成功保存


> [!note] 几个问题：1. 现在是该写model.py了是吧 2. 怎么在代码中使用哪个ResNet-50 3. 要砍掉那个分类头，从第几层开始算头？该怎么砍？4.砍完我们要加一个什么头？什么类型的头：RNN or Dense？要加几层，每层的参数量怎么设计？
> 
> 
> 1. 是的，现在就该写 model.py
> - 放在 src/model.py 最合适。
> - 先只做一件事：EmbeddingModel，输入图像，输出 128-d embedding。
> - 先把前向打通，再接 loss 和训练循环。
> 1. 代码里用哪个 ResNet-50
> - 推荐直接用 torchvision.models.resnet50 + ImageNet 预训练权重（自动下载缓存）。
> - 常见写法思路是：指定 weights=ResNet50_Weights.IMAGENET1K_V2（或 DEFAULT）。
> - 不建议你现在换 timm/其他变体，先用 torchvision 标准版最稳。
> 1. 分类头从哪算，怎么砍
> - 在 torchvision 的 ResNet-50 里，分类头本质是最后的 fc（全连接分类层）。
> - 检索任务里通常“砍头”就是去掉 fc，保留 backbone 到全局池化后的 2048-d 特征。
> - 最实用做法：把 fc 替换成 identity（逻辑上就是移除分类层）。
> - 这样模型输出是每张图一个 2048 向量（再接你自己的 embedding head）。
> 1. 砍完加什么头？RNN 还是 Dense？几层怎么定
> - 用 Dense（MLP/Linear），不要 RNN。  
>   - RNN适合序列（文本/时间），不适合你这种单图特征压缩。
> - 第一版推荐（我建议你先这么做）：
>   - 2048 -> 128 单层线性头
>   - 参数量：2048*128 + 128 = 262,272
>   - 简单、稳定、够你做 baseline。
> - 如果你想稍强一点再上两层：
>   - 2048 -> 512 -> 128（中间可加 ReLU/BN）
>   - 参数量约 1,114,752
>   - 表达力更强，但先别上太复杂，先跑通闭环。
> 我给你的默认建议（P1 起步版）：
> - Backbone: ResNet-50 pretrained
> - Remove: fc
> - Head: Linear(2048, 128)
> - 输出后做 L2 normalize（检索常见，后续 cosine/triplet 更稳）


> [!note] 模型的输入和输出是什么，loss怎么计算
> 
> 
> 模型的输出就是向量。你把三张图分别喂给模型，得到三个向量，然后用这三个向量算 loss：
> 
> ```python
> # 三张图喂给同一个模型，得到三个向量
> embed_anchor   = model(img_anchor)    # 输出：128-d 向量
> embed_positive = model(img_positive)  # 输出：128-d 向量  
> embed_negative = model(img_negative)  # 输出：128-d 向量
> 
> # loss 拿这三个向量算距离
> loss = triplet_loss(embed_anchor, embed_positive, embed_negative)
> 
> # loss 反向传播，更新模型权重
> loss.backward()
> optimizer.step()
> ```
> 
> 你之前问的"模型的输出是 anchor 吗"——我理解你的困惑了。你是在想：模型输出三个向量，我怎么知道哪个是 anchor 哪个是 positive？
> 
> 答案是：**你在喂给模型之前就已经决定好了。** 你采样的时候就说好了"这张是 anchor，这张是 positive，这张是 negative"，然后按顺序喂进去，按顺序拿出来三个向量，直接传给 loss function。模型自己不知道也不需要知道角色是什么，它只管输出向量。
> 
> 一句话：**你喂进去什么角色的图，出来的就是那个角色的向量。顺序是你控制的。**


> [!note] model.py 的结构
> 
> `model.py` 就一个类：`EmbeddingModel`。它做一件事——图片进去，128-d 向量出来。
> 
> 内部两层：
> 
> **第一层：Backbone（ResNet-50）**
> 
> 用 `torchvision` 的预训练权重加载，然后把最后的分类头（`fc` 层）砍掉。ResNet-50 原本最后输出 1000 维（ImageNet 1000 类），你不需要这个，你要的是它倒数第二层的 2048-d 特征。
> 
> **第二层：Embedding Head**
> 
> 接在 backbone 后面，把 2048-d 压到 128-d。三步走：Linear(2048→128) → BatchNorm → L2 Normalize。
> 
> L2 Normalize 是关键——把向量长度归一化为 1，这样后面算 cosine similarity 就等价于算点积，更快也更稳定。
> 
> 结构长这样：
> 
> ```
> 输入图片 (3, 224, 224)
>     ↓
> ResNet-50 backbone（去掉 fc 层）
>     ↓
> 2048-d 特征向量
>     ↓
> Linear(2048 → 128)
>     ↓
> BatchNorm(128)
>     ↓
> L2 Normalize
>     ↓
> 128-d embedding 向量（长度为1）
> ```
> 
> 代码框架是这样，你来填：
> 
> ```python
> import torch
> import torch.nn as nn
> import torchvision.models as models
> import torch.nn.functional as F
> 
> class EmbeddingModel(nn.Module):
>     def __init__(self, embed_dim=128):
>         super().__init__()
>         # 1. 加载预训练 ResNet-50，砍掉 fc 层
>         # 提示：torchvision.models.resnet50(weights=...) 
>         # 然后把 self.backbone.fc 换成 nn.Identity()
>         
>         # 2. 定义 embedding head
>         # Linear(2048, embed_dim) + BatchNorm1d(embed_dim)
>         pass
> 
>     def forward(self, x):
>         # 1. 过 backbone，得到 2048-d
>         # 2. 过 embedding head
>         # 3. L2 normalize（用 F.normalize）
>         # 4. return
>         pass
> ```
> 
> 有一个地方你可能会卡：砍掉 `fc` 层的方式。去查一下 ResNet-50 的结构，看看 `fc` 在哪、怎么替换。这个是你需要自己搞懂的，不是体力活。
> 



# Use ResNet-50


> [!note] 查看ResNet-50的结构
> 
> 
> 两行代码搞定，在终端里跑：
> 
> ```python
> import torchvision.models as models
> print(models.resnet50())
> ```
> 
> 它会把整个网络结构打出来，你会看到最后一层是 `(fc): Linear(in_features=2048, out_features=1000, bias=True)`——那就是你要砍掉的那个。
> 
> 去试试，看完了跟我说你打算怎么换掉它。


> [!note] ResNet50的结构
> 
> 不用全看，你只需要看懂整体骨架就行。把细节折叠掉，结构是这样：
> 
> ```
> ResNet(
>   (conv1)    ← 第一层卷积，输入 3 通道图片
>   (bn1)      ← BatchNorm
>   (relu)     ← 激活函数
>   (maxpool)  ← 下采样
> 
>   (layer1)   ← 残差块组 1，输出 256 通道
>   (layer2)   ← 残差块组 2，输出 512 通道
>   (layer3)   ← 残差块组 3，输出 1024 通道
>   (layer4)   ← 残差块组 4，输出 2048 通道
> 
>   (avgpool)  ← 把空间维度压成 1x1，输出还是 2048 维
>   (fc)       ← Linear(2048 → 1000)  ← 这个就是你要换掉的
> )
> ```
> 
> 每个 `layerX` 里面塞了若干个 `Bottleneck`，那是 ResNet 的残差块，先别管细节，把它当黑盒就行。
> 
> 关键是最后两层：`avgpool` 把图片压成一个 2048-d 向量，`fc` 把这个向量映射到 1000 个类别。
> 
> 你要做的就是**把 `fc` 这层换掉**，换成你自己的 embedding head，这样网络输出的就是 128-d 的向量而不是 1000 个类别的 logit。
> 
> 代码里怎么换？就一行：
> 
> ```python
> backbone = models.resnet50(weights='IMAGENET1K_V1')
> backbone.fc = nn.Identity()  # 把 fc 层变成"什么都不做"
> ```
> 
> `nn.Identity()` 就是透传——输入什么出来什么，相当于把 `fc` 层摘掉了。之后 backbone 的输出就是 2048-d，你再接你自己的 embedding head。
